# US001: プロジェクト作成

## ストーリー

MEMBERユーザーとして、
新しいプロジェクトを作成したい。
それにより、タスクを分類・整理できる。

## 受け入れ基準

- プロジェクト名（必須）と説明（任意）を入力してプロジェクトを作成できる
- 作成したプロジェクトは自分が所有者（owner）となる
- 作成後、プロジェクトIDと作成日時が返却される
- プロジェクト名は1〜100文字

## アンチパターン（これは仕様に含まない）

- プロジェクトの作成時に他のユーザーを所有者として指定することはできない
- 同名のプロジェクトが存在しても作成可能（ユニーク制約なし）
- プロジェクト作成時にタスクを同時に作成することはできない

## テストシナリオ

### 正常系

#### シナリオ1: 名前と説明を指定してプロジェクトを作成

- **Given（前提）**: ユーザーがログイン済み（X-User-Id: 123）
- **When（操作）**: POST /projects に `{"name": "新規プロジェクト", "description": "説明文"}` を送信
- **Then（結果）**:
  - HTTPステータス 201 が返る
  - レスポンスに `id`, `name`, `description`, `ownerId: 123`, `createdAt` が含まれる

#### シナリオ2: 名前のみでプロジェクトを作成

- **Given（前提）**: ユーザーがログイン済み
- **When（操作）**: POST /projects に `{"name": "プロジェクトA"}` を送信
- **Then（結果）**:
  - HTTPステータス 201 が返る
  - `description` は null として返る

### 異常系

#### シナリオ3: 名前が空の場合

- **Given（前提）**: ユーザーがログイン済み
- **When（操作）**: POST /projects に `{"name": ""}` を送信
- **Then（結果）**:
  - HTTPステータス 400 が返る
  - エラーコード `TASK_PROJECT_VALIDATION_ERROR` が返る

#### シナリオ4: 名前が100文字を超える場合

- **Given（前提）**: ユーザーがログイン済み
- **When（操作）**: POST /projects に 101文字の `name` を送信
- **Then（結果）**:
  - HTTPステータス 400 が返る
  - エラーコード `TASK_PROJECT_VALIDATION_ERROR` が返る

#### シナリオ5: 認証なしでアクセス

- **Given（前提）**: X-User-Id ヘッダがない
- **When（操作）**: POST /projects を送信
- **Then（結果）**:
  - HTTPステータス 401 が返る
