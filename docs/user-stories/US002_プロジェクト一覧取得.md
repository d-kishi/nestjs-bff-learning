# US002: プロジェクト一覧取得

## ストーリー

MEMBERユーザーとして、
プロジェクトの一覧を取得したい。
それにより、管理しているプロジェクトを確認できる。

## 受け入れ基準

- プロジェクト一覧をページネーション付きで取得できる
- 所有者（ownerId）でフィルタリングできる
- 各プロジェクトの基本情報（id, name, description, ownerId, createdAt, updatedAt）が取得できる
- デフォルトでは1ページあたり20件取得

## アンチパターン（これは仕様に含まない）

- 一覧取得時に各プロジェクト配下のタスク情報は含まない（詳細取得で対応）
- 自分が所有していないプロジェクトも一覧に含まれる（Phase 1では権限フィルタなし）
- ソート順の指定はできない（作成日時の降順固定）

## テストシナリオ

### 正常系

#### シナリオ1: プロジェクト一覧を取得

- **Given（前提）**: プロジェクトが3件登録されている
- **When（操作）**: GET /projects を送信
- **Then（結果）**:
  - HTTPステータス 200 が返る
  - `data` に3件のプロジェクトが含まれる
  - `meta.total` が 3 である

#### シナリオ2: ページネーションで2ページ目を取得

- **Given（前提）**: プロジェクトが25件登録されている
- **When（操作）**: GET /projects?page=2&limit=10 を送信
- **Then（結果）**:
  - `data` に10件のプロジェクトが含まれる
  - `meta.page` が 2、`meta.limit` が 10、`meta.total` が 25

#### シナリオ3: 所有者でフィルタリング

- **Given（前提）**: ユーザー123のプロジェクトが2件、ユーザー456のプロジェクトが1件
- **When（操作）**: GET /projects?ownerId=123 を送信
- **Then（結果）**:
  - `data` に2件のプロジェクトが含まれる
  - すべてのプロジェクトの `ownerId` が 123

#### シナリオ4: プロジェクトが0件の場合

- **Given（前提）**: プロジェクトが登録されていない
- **When（操作）**: GET /projects を送信
- **Then（結果）**:
  - HTTPステータス 200 が返る
  - `data` は空配列 `[]`
  - `meta.total` が 0

### 異常系

#### シナリオ5: 不正なページ番号

- **Given（前提）**: プロジェクトが存在する
- **When（操作）**: GET /projects?page=-1 を送信
- **Then（結果）**:
  - HTTPステータス 400 が返る
  - エラーコード `TASK_PROJECT_VALIDATION_ERROR`
