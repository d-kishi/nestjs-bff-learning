# docs/sessions/daily/

日次のセッション記録を格納するディレクトリです。

## ファイル命名規則

```
YYYY-MM-DD.md

例:
2025-01-15.md
2025-01-16.md
```

## テンプレート

```markdown
# YYYY-MM-DD セッション記録

## セッション概要

| 項目 | 内容 |
|------|------|
| 目的 | [今日のセッション目的] |
| 作業特性 | 新機能実装 / 機能拡張 / 品質改善 / 調査分析 |

## 完了事項

- [完了した作業1]
- [完了した作業2]

## 技術的知見・学び

- [新しく学んだこと、気づき]

## 発生した問題と解決

### 問題1: [問題の概要]
- **原因**: [原因]
- **解決策**: [解決方法]

## 未完了・継続事項

- [ ] [継続が必要な作業]

## 次回への申し送り

- [次回セッションで最初に確認すべきこと]
- [注意点・制約]
```

## 記録のポイント

- 完了事項は具体的に（ファイル名、変更内容）
- 技術的知見は後で振り返れる粒度で
- 問題と解決は再発防止のため詳細に
