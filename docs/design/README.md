# 設計ドキュメント

Phase 1以降の詳細設計を管理するディレクトリ。

## ファイル構成

| ファイル | 内容 |
|---------|------|
| task-service-entities.md | task-serviceのエンティティ詳細設計 |
| task-service-api.md | task-serviceのAPI設計 |
| user-service-entities.md | user-serviceのエンティティ詳細設計（Phase 2） |
| user-service-api.md | user-serviceのAPI設計（Phase 2） |

## 設計ドキュメントの役割

- **企画書 (`docs/project-plan.md`)**: 全体方針、アーキテクチャ概要
- **設計ドキュメント (`docs/design/`)**: 実装に必要な詳細設計
- **ユーザーストーリー (`docs/user-stories/`)**: 機能要件、テストシナリオ

設計ドキュメントは、企画書の方針に基づき、実装時に必要な具体的な情報を記載する。
