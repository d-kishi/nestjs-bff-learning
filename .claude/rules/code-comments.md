# コードコメント規約

## 目的

このプロジェクトは**学習目的**のため、実装の意図・設計判断を後から理解できるよう、通常より丁寧なコメントを記載する。

## 基本ルール

- **言語**: 日本語
- **対象**: 全ファイル（Controller, Service, Repository, DTO, Entity）

## 記載すべき内容

1. **What**: 何をしているか（簡潔に）
2. **Why**: なぜこの実装・設計を選んだか（最重要）
3. **How**: 複雑な場合のみ、どのように動作するか

## 重点ポイント

- クラス・メソッドには責務と設計意図を記載
- DTOにはバリデーション設計の理由を記載
- 複雑なロジックには「なぜこの実装か」を記載
- NestJSのデコレータ・ガード・パイプの適用理由を記載

## 記載不要

- 自明な処理の説明
- TypeScriptの型情報で明らかなこと
