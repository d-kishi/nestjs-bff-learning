# セッション開始Command

**目的**: セッション開始時の状況確認・目標設定を実行
**対象**: 全セッション

## コマンド実行内容

### 1. 前回セッション情報の確認（必須）

- [ ] `docs/sessions/project-status.md` を読み込み、現在のPhase・状況を把握
- [ ] `docs/sessions/daily/` の直近ファイルを確認（前回の申し送り事項）
- [ ] `docs/sessions/task-checklist.md` で中長期タスクの状況を把握

### 2. 基本状況確認（必須）

- [ ] `CLAUDE.md` - プロジェクト概要・技術構成確認
- [ ] 必要に応じて `docs/project-plan.md` の該当Phaseを確認

### 3. セッション目的の明確化

- [ ] ユーザーとセッション目的を確認
- [ ] 作業特性を判断（新機能実装 / 機能拡張 / 品質改善 / 調査分析）
- [ ] 必要に応じてADR・設計書の読み込み要否を判断

### 4. セッション目標設定・TodoList初期化

- [ ] 明確なセッション目標を設定
- [ ] TodoWriteツールでタスク管理を初期化
- [ ] 作業の優先順位を確認

## 実行後アクション

✅ 全確認完了 → 本作業開始
⚠️ 情報不足 → 追加情報収集後に本作業開始
❌ 目的不明確 → ユーザーとの目的再確認

## ユーザーへの報告形式

```markdown
## セッション開始確認

### 現在の状況
- Phase: [現在のPhase]
- 前回の成果: [前回完了事項の要約]
- 申し送り: [前回からの申し送り事項]

### 本日の目標
- [目標1]
- [目標2]

### 確認事項
- [ユーザーに確認したいこと]
```

## 自動実行トリガー

- ユーザー発言: "セッションを開始します"
- ユーザー発言: "セッション開始"
- ユーザー発言: "/session-start"
